Прикрепляю решение задания ниже.
1) Анализ конкурентов
Для понимания процесса проведен анализ сервисов "Островок" и "Яндекс.Путешествия", чтобы понять их преимущества и недостатки, наша реализация постарается взять все лучшее из этих двух сервисов.
- Островок: делает упор на детальную фильтрацию (тип кровати, завтраки, расстояние до центра) и пользовательский контент (реальные фото в отзывах). Это помогает пользователю сделать осознанный выбор.
- Яндекс Путешествия: выделяется за счет экосистемы. Возможность оплаты баллами и автозаполнение данных из аккаунта значительно ускоряют процесс бронирования.
Итог: Моя реализация будет сочетать гибкие фильтры (как на «Островке») и максимально быстрое оформление через подтягивание данных из профиля пользователя (как в «Яндексе»).
2) Глоссарий
- Гость: пользователь, который ищет номер и совершает бронирование.
- Отель: объект размещения, предоставляющий услуги проживания.
- Бронирование: фиксация номера за пользователем на конкретные даты.
- Ваучер: документ, подтверждающий бронь и оплату.
3) Бизнес-цели
- Автоматизировать подтверждение бронирования.
- Сократить время оформления заказа до 3 минут.
- Исключить ошибки, связанные с овербукингом (двойным бронированием одного номера).
4) Описание бизнес-процесса
1. Гость выбирает город и даты проживания.
2. Система подбирает доступные варианты и выводит список.
3. Гость выбирает конкретный номер.
4. Система запрашивает данные гостя и данные для оплаты.
5. Средства блокируются, запись создается в базе данных, Гость получает подтверждение (ваучер).
5) User Story
«Как зарегистрированный пользователь, я хочу, чтобы мои данные автоматически подставлялись в форму бронирования, чтобы не вводить их каждый раз вручную».
6) Ограничения
- Интеграция минимум с тремя агрегаторами отелей.
- Безопасная передача данных карт (соответствие PCI DSS).
- Бронирование доступно не позднее, чем за 1 час до заезда.
7) Критерии готовности (DoR)
- Подготовлен дизайн-макет страницы оформления.
- Согласован список обязательных полей для формы.
- Есть документация API для получения списка номеров.
8) Критерии завершенности (DoD)
- Данные успешно сохранены в БД со статусом «Ожидает оплаты».
- Пользователю отправлено уведомление (email) с номером брони.
- Проведены тесты сценария успешной оплаты.
